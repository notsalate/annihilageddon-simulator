# Крутагидон 2 Simulation Codex

Локальный проект для разработки deterministic-симулятора настольной игры "Эпичные схватки боевых магов: Крутагидон 2".

Главная цель - прогонять десятки тысяч партий, изучать закономерности игры, сравнивать стратегии, оценивать влияние стартовой барахолки, колод и решений ботов.

## Текущий статус

Проект находится на этапе спецификации.

Код движка еще не создан. Сейчас зафиксированы:

- границы первой версии симулятора;
- контракты для агентов;
- правила импорта карт;
- подход к будущему TypeScript-ядру.
- v0-oriented canon draft правил.

## Основные решения

- Симулятор headless: UI не является целью.
- Основной интерфейс - консольные команды.
- Ядро будет написано на TypeScript.
- TypeScript должен быть максимально строгим с самого начала.
- Симуляции должны быть воспроизводимыми через seeded RNG.
- `Math.random()` в ядре запрещен.
- Карты хранятся как JSON-данные.
- Поведение карт реализуется через явные typed handlers, а не через парсинг текста карты во время партии.
- Боты выбирают только из легальных действий, а правила применяет engine.
- Подробный event log нужен для отладки одной партии.
- Массовые прогоны по умолчанию должны сохранять краткий summary, а не полный лог.

## Первая версия

v0 должна дать первую runnable-симуляцию, но не обязана ждать полной оцифровки всех карт.

В v0 входят:

- 2 игрока;
- базовые стартовые колоды по правилам;
- отдельные экземпляры карт для каждого игрока;
- минимальный импортированный набор русских карт;
- основная колода;
- колода Легенд;
- жетоны дохлых колдунов;
- барахолка;
- добор, рука, сброс, перемешивание;
- покупка и получение карт в сброс;
- карты под контролем;
- постоянки;
- реальные условия конца партии;
- подсчет победителя по ПО.

Полная оцифровка 100+ карт основной колоды и колоды Легенд не блокирует v0.

## Условия конца партии

Партия заканчивается, если:

- закончились жетоны дохлых колдунов;
- закончилась основная колода;
- закончилась колода Легенд.

`maxTurns` может использоваться только как техническая защита от бага или вечного цикла.

## Определение победителя

Победитель определяется так:

1. Больше всего ПО.
2. Если ничья - больше всего карт Легенд.
3. Если снова ничья - меньше дохлых колдунов.
4. Если снова ничья - настоящая ничья.

ПО карты должно храниться отдельным числовым полем.

При scoring учитываются карты игрока:

- в руке;
- в колоде;
- в сбросе;
- под контролем.

## Модель карт

Проект различает три сущности:

- Card definition - уникальное описание карты.
- Deck composition - сколько копий каких карт входит в колоду.
- Card instance - конкретная копия карты в конкретной партии.

Один JSON-файл описывает одну уникальную карту. Количество копий хранится отдельно в составе колоды.

Формат `cardId`:

```text
esw2_dbg__stable_catalog_id
```

Пример:

```text
esw2_dbg__main_001
```

`cardId` не должен зависеть от OCR-названия карты. Русское название хранится отдельно как видимое поле.

## Импорт карт

Полный импорт карт будет идти отдельным пайплайном:

1. Пользователь добавляет изображения карт в `assets/cards/raw/*.{webp,png,jpg,jpeg}`.
2. OCR agent создает `data/import/card-texts/<card-id>.md`.
3. Card JSON agent создает `data/import/card-drafts/<card-id>.json`.
4. Engine Mapping agent создает финальные runtime-поля для `data/cards/<card-id>.json`.

Первый OCR-этап извлекает только видимые русские данные с изображений. Он не ищет английские названия, не переводит текст и не назначает engine effects.

## Планируемая структура

Структура будет уточнена при создании TypeScript-проекта.

Ожидаемые директории:

```text
docs/
data/
  cards/
  decks/
  import/
src/
tests/
.scratch/
```

Не создавать эти директории заранее без задачи, если для них еще нет содержимого.

## Команды

Проект использует минимальный headless TypeScript scaffold.

Посмотреть доступные npm scripts:

```powershell
npm run
```

Проверить типы:

```powershell
npm run typecheck
```

Запустить тесты:

```powershell
npm test
```

## Рабочий процесс

Рекомендуемый порядок ближайших работ:

1. Расширить `docs/rules-canon.md` до full global mechanics canon.
2. Продолжить OCR первого батча карт.
3. Создать минимальный TypeScript scaffold.
4. Реализовать engine v0 на маленьком импортированном наборе русских карт.
5. Запустить воспроизводимые партии с seeded RNG.
6. После этого подключать полный импорт карт из изображений.
