# Сгенерировать базовые card cluster decisions и matrix

## Parent

- `.scratch/krutagidon-card-runtime-clusters/PRD.md`

## What to build

Создать базовый import-side workflow для кластеризации card drafts: structured decisions JSON как ручной источник решений и autogenerated matrix markdown как отчёт.

Срез должен быть end-to-end: npm command читает все card drafts, создаёт или дополняет decisions через явный флаг, генерирует matrix, и локальный `.scratch` контракт объясняет, какие файлы править руками, а какие нельзя.

Этот issue не вводит все fullRuntime guardrails. Он делает минимально полезный generated workflow, который следующие issues смогут ужесточить.

## User stories covered

- 9. Structured decisions file.
- 10. Autogenerated matrix.
- 11. Commands to generate and refresh reports.
- 12. Missing decisions detected.
- 14. Generated planning is based on import drafts/source text.
