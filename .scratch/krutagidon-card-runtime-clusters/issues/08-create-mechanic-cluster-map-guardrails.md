# Ввести mechanic cluster map и guardrails

## Parent

- `.scratch/krutagidon-card-runtime-clusters/PRD.md`

## What to build

Создать проверяемую основу для Block C planning: отдельный human-readable файл `mechanic-clusters.md`, где каждый финальный runtime cluster описывается заголовком `## cluster-id`.

Срез должен добавить guardrails между manual decisions и mechanic cluster map, но не должен назначать все карты по кластерам и не должен реализовывать runtime cards.

## User stories covered

- 10. Autogenerated matrix does not rot.
- 11. Commands to generate and refresh reports.
- 12. Missing decisions detected.
- 15. Block C card implementation deferred until Block A and Block B are finished.
