# Объединить mechanics inventory batches

## Parent

- `.scratch/krutagidon-card-runtime-clusters/PRD.md`

## What to build

Объединить результаты пяти inventory batches в единый planning synthesis для будущей финальной кластеризации.

Срез должен нормализовать candidate mechanics, убрать дубли и подготовить явные cardId batches для следующих clustering issues. Это всё ещё planning работа: не назначать финальный `clusterId` всем картам и не реализовывать runtime cards.

## User stories covered

- 10. Autogenerated matrix does not rot.
- 12. Missing decisions detected.
- 14. Generated runtime cards based on import drafts/source text.
- 15. Block C card implementation deferred until Block A and Block B are finished.
